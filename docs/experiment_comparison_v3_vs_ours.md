# 실험 비교 분석: v3(박민서) vs 우리 팀(이정한)

작성일: 2026-05-20  
비교 대상: `docs/v3 학습결과` vs `docs/5.15 ~ 5.18 모델학습 결과 보고서`

---

## 1. 핵심 수치 비교

| 항목 | 박민서 (v3) | 우리 팀 (exp_008~010) |
|---|---|---|
| 총 데이터 행 수 | 6,077행 | ~46,000행 (train+valid+test) |
| 불균형 비율 | **1.1:1** (거의 균형) | **124:1** (극심한 불균형) |
| 모델 | XGBoost | RF + signed_log1p |
| Test F1 | **0.7482** | 0.3529 |
| Test PR-AUC | **0.8874** | 0.2876 |
| Test Recall | 0.6546 | 0.3750 |
| Test Precision | 0.8729 | 0.3333 |

수치만 보면 박민서 실험이 압도적으로 좋아 보인다. 그러나 두 실험은 근본적으로 다른 문제를 풀고 있다.

---

## 2. 차이점 상세 분석

### 2-1. 라벨 정의 — 가장 중요한 차이

| | 박민서 (v3) | 우리 팀 |
|---|---|---|
| label=1 정의 | 상폐 기업의 **모든 연도·분기 데이터** | 상폐연도 기준 **1년 전 데이터만** |
| 예측 문제 | "이 기업이 상폐된 기업인가?" | "이 기업이 1년 내 상폐될 것인가?" |
| 운영 시나리오 적합성 | 낮음 | 높음 |

**박민서 방식의 문제:**  
상폐 기업의 모든 기간 데이터(상폐 5년 전, 10년 전 포함)에 label=1을 부여한다. 예를 들어 2025년에 상폐된 기업의 2015년 데이터에도 label=1이 붙는다.

- 근거: v3 보고서 §2 "라벨 기준: `raw/healthy`는 `label=0`, `raw/delisted`는 `label=1`"
- 이는 **기업 분류 문제**이지, **조기경보 예측 문제가 아니다**

**우리 방식 (fixed_N1):**  
"상폐연도 - 1년"에 해당하는 데이터만 label=1로 부여한다. 나머지 시점은 해당 기업이 상폐 기업이더라도 label=0이다.

- 근거: 5.15 보고서 §0 "N1은 상장폐지 1년 전 연도의 모든 분기를 양성으로 정의"
- 실제 운영 목적("지금 이 재무데이터를 보고 1년 내 상폐를 예측할 수 있는가")에 직접 대응

---

### 2-2. 불균형 비율 — 성능 착시의 원인

| | 박민서 (v3) | 우리 팀 |
|---|---|---|
| label=0 | 3,229개 | ~44,000개 (train 기준) |
| label=1 | 2,848개 | 283개 (train 기준) |
| 비율 | **1.1:1** | **124:1** |

**착시가 발생하는 이유:**  
박민서 데이터에서 imbalance가 1.1:1인 것은 라벨 정의의 직접적인 결과다. 상폐 기업 수십 개의 수년치 분기 데이터를 모두 양성으로 쌓으면 2,848개가 된다. 균형 잡힌 데이터에서 F1 0.75 이상은 어렵지 않다.

실제 코스닥 시장에서 연간 상폐 비율은 수 퍼센트 수준이며, 분기 단위로 보면 124:1이 현실에 가깝다.

- 근거: 5.15 메타 표 — fixed_N1 train imbalance_ratio: 124.4
- 불균형 문제를 그대로 안고 가는 것이 실제 문제를 정직하게 푸는 것이다

---

### 2-3. 데이터 분할 방식

| | 박민서 (v3) | 우리 팀 |
|---|---|---|
| 방식 | **Group Split** (종목코드 기준) | **Time Split** (연도 기준) |
| 기업 중복 | 없음 | 있음 (88~100%) |
| 시계열 순서 보장 | **보장 안 됨** | 보장됨 (train→valid→test) |

**Group Split의 한계:**  
박민서 방식은 같은 종목코드가 train/valid/test에 중복 등장하지 않아 기업 암기는 방지한다. 그러나 시계열 순서를 보장하지 않아 2024년 데이터가 train에, 2015년 데이터가 test에 들어갈 수 있다.

- 근거: v3 보고서 §4, split 결과 — train/valid/test 연도가 아닌 종목코드 기준 분리
- 미래 정보가 과거 예측에 활용될 수 있는 **시계열 누수(look-ahead leakage)** 가능성이 있다

**우리 방식:**  
train 2015–2022 → valid 2023 → test 2024 순서를 고정한다. 모델이 미래 정보를 볼 수 없다. 기업 중복 문제는 Optuna 튜닝 시 GroupKFold(stock_code 기준)를 내부 CV에 적용해 부분적으로 완화한다.

- 근거: 5.2 설계 §4-2 "Optuna 튜닝은 train 데이터만 사용, valid set은 격리"

---

### 2-4. 피처 구성

| | 박민서 (v3) | 우리 팀 |
|---|---|---|
| 피처 수 | **64개** | 33개 |
| 재무비율 | O | O |
| 결측 flag (`_missing`) | O | X |
| 보고기간 one-hot | O (Q1/H1/Q3/ANNUAL) | X |
| 매크로 변수 | X | O (credit_spread, VIX 등 6종) |
| 원시 금액 변수 | X | O (유형자산 등 5종) |

박민서는 `_missing` flag 피처를 포함한다. 상폐 기업이 구조적으로 특정 항목을 공시하지 않는 패턴이 있다면 이 피처가 강력한 신호가 될 수 있다. 그러나 이는 "상폐 기업이라는 사실"이 결측 패턴에 이미 반영된 것일 수 있어 라벨 누수와 유사한 효과를 낼 위험이 있다.

## 3. 라벨 정의 효과 검증 실험 (exp_label_scheme)

"라벨 정의가 다르기 때문이다"는 주장을 수치로 검증하기 위해 직접 실험을 설계했다.

### 3-1. 실험 설계

모델·피처·Time Split을 모두 고정하고 **라벨 정의만** 바꿨다.

```
고정 조건 (두 실험 동일)
  - 모델:     RF (n_estimators=200, max_depth=10, min_samples_leaf=5)
  - 피처:     33개 + signed_log1p FE
  - 데이터:   fixed_N1 / exp-A
  - 분할:     Time Split (train 2015-2022 / valid 2023 / test 2024)

변경 조건 (하나만 다름)
  - fixed_N1_label: 상폐연도 1년 전 데이터만 label=1
  - minseo_label:   상폐 기업의 모든 데이터를 label=1
```

라벨을 바꾸면 양성 샘플 수가 달라진다.

| split | fixed_N1_label 양성 | minseo_label 양성 | 배율 |
|---|---|---|---|
| train | 283개 (0.8%) | 2,103개 (5.9%) | 7.4배 |
| valid | 39개 (0.8%) | 129개 (2.6%) | 3.3배 |
| test | 56개 (1.1%) | 93개 (1.8%) | 1.7배 |

실험 결과는 `results/exp_label_scheme/fixed_N1_label.json`, `results/exp_label_scheme/minseo_label.json`에 저장됐다.

### 3-2. 실험 결과

| 라벨 방식 | Valid PR-AUC | Test PR-AUC | Test Precision | Test Recall | Test F1 |
|---|---|---|---|---|---|
| fixed_N1_label (우리) | 0.1190 | **0.2876** | 0.3333 | 0.3750 | 0.3529 |
| minseo_label (박민서) | **0.4222** | 0.2341 | 0.1577 | 0.3763 | 0.2222 |
| Δ (minseo − fixed_N1) | **+0.3032** | **−0.0535** | −0.1756 | +0.0013 | −0.1307 |

### 3-3. 해석

**Valid PR-AUC가 0.12 → 0.42로 3.5배 상승한다.**

minseo 방식으로 라벨을 붙이면 train에서 상폐 기업 106개의 수년치 데이터가 모두 양성이 된다. 모델은 "이 기업들의 재무 패턴"을 학습하고, valid(2023)에 같은 기업 데이터가 등장하면 잘 알아본다. 박민서 보고서의 validation PR-AUC 0.83, F1 0.79가 나오는 구조적 이유가 바로 이것이다.

**Test PR-AUC는 0.2876 → 0.2341로 오히려 하락한다.**

test 양성 수가 56→93개로 늘었는데도(양성이 많으면 PR-AUC가 올라가기 쉬운 조건임에도 불구하고) PR-AUC가 내려갔다. 이는 모델이 학습한 "상폐 기업 유형 패턴"이 실제 미래 시점의 조기경보 신호로 전이되지 않는다는 직접적인 수치 근거다.

**Precision이 절반으로 줄었다 (0.3333 → 0.1577).**

경보를 더 많이 울리지만 맞는 비율이 낮아졌다. minseo 방식 모델은 실제 운영 시 오탐 부담이 2배 이상 늘어난다.

### 3-4. 실험의 한계

이 실험이 박민서 환경을 완전히 재현하지는 않는다.

| 항목 | 박민서 실제 | 우리 minseo_label 재현 |
|---|---|---|
| test 양성 비율 | **45.3%** | 1.8% |
| 총 데이터 규모 | 6,077행 | 45,854행 |

박민서 환경의 45% 양성 비율은 재현하지 못했다. 따라서 박민서의 F1 0.75 수준을 수치로 직접 재현해 보여주지는 못한다. 그러나 이 실험이 보여주는 것은 수치가 아닌 **패턴**이다: 라벨 정의를 바꾸면 valid 지표가 크게 부풀고, test에서는 오히려 역효과가 난다. 박민서 환경에서는 양성 비율 차이까지 더해져 이 효과가 더 크게 나타났을 것으로 추정된다.

## 4. 높은 성능 수치의 종합 해석

박민서 실험의 test PR-AUC 0.8874, F1 0.7482는 다음 세 가지 요인의 결합으로 설명된다.

| 요인 | 영향 |
|---|---|
| 라벨 정의 (기업 분류) | 상폐 기업 자체의 재무 패턴(규모, 섹터 등)을 학습 → 쉬운 문제 |
| 균형 잡힌 데이터 (1.1:1) | F1, Recall 지표가 자동으로 높게 나오는 구조 |
| 시계열 누수 가능성 | 미래 데이터로 과거를 예측하는 상황이 발생할 수 있음 |

반면 우리 실험의 PR-AUC 0.2876은 **실제 조기경보 문제의 어려움을 그대로 반영**한 것이다. 1년 내 상폐를 재무비율만으로 예측하는 것은 본질적으로 어려운 문제이며, 0.29 수준은 random baseline(= 양성 비율 ≈ 0.008) 대비 약 36배 높은 수치다.

## 5. 어떤 접근이 목적에 맞는가

프로젝트의 목표는 **상장폐지 조기경보 시스템**이다. 운영 시나리오는 다음과 같다.

> "현재 시점에서 수집된 재무 데이터를 보고, 이 기업이 1년 내 상폐될 가능성을 예측한다."

이 목적에 맞는 실험 설계 요건:

| 요건 | 박민서 (v3) | 우리 팀 |
|---|---|---|
| 라벨이 미래 시점 상폐 여부를 예측하는가 | X | **O** |
| 실제 불균형 비율을 반영하는가 | X | **O** |
| 시계열 순서를 보장하는가 | X | **O** |
| 평가 지표가 불균형 문제에 적합한가 | 부분적 (F1 사용) | **O** (PR-AUC primary) |

---

## 6. 박민서 실험에서 참고할 수 있는 점

비판만 하는 것은 공정하지 않다. 박민서 실험에서 우리가 검토해볼 수 있는 아이디어도 있다.

- **`_missing` flag 피처**: 결측 패턴이 상폐 신호가 될 수 있다. 단, 라벨 정의가 올바른 상태에서 실험해야 의미 있다
- **보고기간 one-hot**: Q1/H1/Q3/ANNUAL 분기 특성이 피처로 유용할 수 있다
- **Group Split의 아이디어**: 기업 암기 방지를 위한 GroupKFold는 우리도 Optuna 내부 CV에 이미 적용 중
