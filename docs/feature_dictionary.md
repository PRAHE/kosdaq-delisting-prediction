## 1. 성장성 (Growth)

| 피처명 | 계산식 | 단위 | 계산 위치 | 비고 |
|---|---|---|---|---|
| 총자산증가율 | (기말총자산 - 기초총자산) / 기초총자산 × 100 | % | ratio_calculator.py | BS frmtrm 기반, 안정적 |
| 유동자산증가율 | (기말유동자산 - 기초유동자산) / 기초유동자산 × 100 | % | ratio_calculator.py | BS frmtrm 기반, 안정적 |
| 매출액증가율 | (당기매출액 - 전년동기매출액) / \|전년동기매출액\| × 100 | % | build_master_dataset.py | YoY 방식 (전년 동기 조인) |
| 순이익증가율 | (당기순이익 - 전년동기순이익) / \|전년동기순이익\| × 100 | % | build_master_dataset.py | YoY 방식 (전년 동기 조인) |
| 영업이익증가율 | (당기영업이익 - 전년동기영업이익) / \|전년동기영업이익\| × 100 | % | build_master_dataset.py | YoY 방식 (전년 동기 조인) |

> **IS 증가율 3개 YoY 교체 이유**  
> DART 분기/반기 보고서는 IS frmtrm(전기값)을 비워 공시하는 경우가 많아  
> frmtrm 기반 계산 시 Q1/H1/Q3 결측률 92%+. 전년 동기 직접 조인으로 13~14%로 개선.  
> 분모에 abs() 적용: 전기가 음수일 때 부호 왜곡 방지.

## 2. 수익성 (Profitability)

| 피처명 | 계산식 | 단위 | 판별력 |
|---|---|---|---|
| 매출액순이익률 | 순이익 / 매출액 × 100 | % | ✅ 높음 (정상 중앙값 2.9 vs 상폐 -17.4) |
| 매출총이익률 | 매출총이익 / 매출액 × 100 | % | ✅ 높음 (정상 21.3 vs 상폐 15.3) |
| 자기자본순이익률 | 순이익 / 자기자본 × 100 (ROE) | % | ✅ 높음 (정상 1.2 vs 상폐 -3.9) |

> **주의사항**  
> 수익성 계열은 왜도 -206~-172, 첨도 20,000~43,000으로 극단적 분포.  
> 상폐 직전 기업의 재무 악화가 극단값 형태로 나타난 것이므로 **제거 금지**.  
> clipping 경계값 집중은 0%대로 문제 없음. Winsorize/RobustScaler 실험 예정.

## 4. 안정성 (Stability)

| 피처명 | 계산식 | 단위 | clipping 범위 |
|---|---|---|---|
| 부채비율 | 부채 / 자기자본 × 100 | % | 0 ~ 2,000 |
| 유동비율 | 유동자산 / 유동부채 × 100 | % | 0 ~ 2,000 |
| 자기자본비율 | 자기자본 / 총자산 × 100 | % | -500 ~ 100 |
| 당좌비율 | (유동자산 - 재고자산) / 유동부채 × 100 | % | 0 ~ 2,000 |
| 비유동자산장기적합률 | 비유동자산 / 장기차입금 | 배 | 0 ~ 2,000 |
| 순운전자본비율 | (유동자산 - 유동부채) / 총자산 × 100 | % | -500 ~ 100 |
| 차입금의존도 | (단기차입금 + 장기차입금 + 사채) / 총자산 × 100 | % | 0 ~ 100 |
| 현금비율 | 현금및현금성자산 / 유동부채 × 100 | % | 0 ~ 2,000 |
| 유형자산 | 유형자산 절대값 | 원 | IQR×3 |
| 무형자산 | 무형자산 절대값 | 원 | IQR×3 |
| 무형자산상각비 | CF 무형자산상각비 | 원 | IQR×3 |
| 유형자산상각비 | CF 유형자산감가상각비 | 원 | IQR×3 |
| 감가상각비 | 유형자산상각비 + 무형자산상각비 | 원 | IQR×3 |

> **차입금의존도 특이사항**  
> zero-inflated 분포 확인 (하한 0에 5.8% 집중).  
> 모델링 단계에서 `차입금의존도 > 0` 이진 파생변수 추가 실험 예정 (이슈 #5).

> **자산 규모 피처 (유형자산, 무형자산) 특이사항**  
> 절대값이라 산업별 규모 차이를 반영할 수 있어 상폐 신호보다 기업 구조 차이를  
> 나타낼 가능성 있음. 총자산 대비 비율 변환 실험 예정 (이슈 #4).

## 7. 전처리 파이프라인 요약

### 결측치 처리 순서 (build_h_datasets.py Preprocessor)

1. **ffill** - 기업 내 이전 분기값으로 채움 (구조적 결측 처리)
2. **CF=0** - 감가상각비 계열 결측은 0으로 처리
3. **섹터·분기 중앙값 보간** - 그룹 내 중앙값으로 채움
4. **전체 중앙값 fallback** - 그래도 결측이면 전체 중앙값
5. **clipping** - DOMAIN_RULES 기반 이상치 제거

### 실험 설정 (Winsorize / RobustScaler)

| 실험 | Winsorize | RobustScaler | 저장 경로 |
|---|---|---|---|
| baseline | ❌ | ❌ | `H{N}/baseline/` |
| exp-A | ✅ | ❌ | `H{N}/exp-A/` |
| exp-B | ❌ | ✅ | `H{N}/exp-B/` |
| exp-C | ✅ | ✅ | `H{N}/exp-C/` |

Winsorize 대상: 수익성 계열 5개 (매출액순이익률, 자기자본순이익률, 총자본영업이익률, 총자본순이익률, 매출총이익률)  
Winsorize 범위: 양쪽 1% (train 기준 fit, valid/test는 transform만)

### 1차 실험 구간: H14, H16, H18

| H | train 양성 | valid 양성 | test 양성 | test 행수 | 불균형 |
|---|---|---|---|---|---|
| 10 | 288 | 21 | 48 | 5,460 | 114:1 |
| 12 | 341 | 27 | 69 | 5,460 | 96:1 |
| **14** | **385** | **35** | **81** | **5,460** | **82:1** ✅ |
| **16** | **453** | **48** | **75** | **4,056** | **69:1** ✅ |
