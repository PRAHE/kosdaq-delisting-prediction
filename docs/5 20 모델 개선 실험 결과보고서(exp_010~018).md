# 📌 exp_010: Threshold 전략 비교 실험

## 0. 실험 목적

best 모델(RF + fixed_N1 + exp-A + signed_log1p) 위에서 threshold를 바꾸면 Recall을 얼마나 끌어올릴 수 있는지, 그 대가(Precision 하락)는 어느 수준인지 정량화한다.

상폐 조기경보 시스템 특성상 FN(놓친 상폐) 비용이 FP(오탐) 비용보다 크므로 Recall 편향 전략이 현실적 요구에 부합하는지 확인하는 것이 목적이다.

## 1. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A (train 2015–2022 / valid 2023 / test 2024) |
| Model | RF (n_estimators=200, max_depth=10, min_samples_leaf=5, max_features=sqrt) |
| FE | signed_log1p |
| Threshold 기준 | valid set |
| Test 양성 수 | 56개 |

| 전략 | 설명 |
|------|------|
| F1 | valid F1 최대화 (현행 baseline) |
| F2 (β=2) | valid F2 최대화 (Recall 2배 가중) |
| Recall≥70% | valid Recall ≥ 70% 구간에서 Precision 최대화 |
| Recall≥80% | valid Recall ≥ 80% 구간에서 Precision 최대화 |

## 2. 결과

| 전략 | Threshold | Valid Recall | Test PR-AUC | Test Recall | Test Precision | Test F1 | 경보수 |
|------|-----------|-------------|-------------|-------------|----------------|---------|--------|
| F1 (baseline) | 0.097 | 0.282 | 0.2876 | **0.375** | **0.333** | **0.353** | 63 |
| F2 (β=2) | 0.062 | 0.359 | 0.2876 | 0.411 | 0.204 | 0.272 | 113 |
| Recall≥70% | 0.015 | 0.718 | 0.2876 | 0.571 | 0.046 | 0.085 | 696 |
| Recall≥80% | 0.009 | 0.821 | 0.2876 | 0.696 | 0.038 | 0.072 | 1,026 |

*Test 전체 양성: 56개*

## 3. 해석

**PR-AUC는 어떤 전략에서도 0.2876으로 동일하다.** Threshold는 PR curve 위의 운영점을 선택할 뿐, 모델의 랭킹 능력 자체를 바꾸지 않는다. Recall을 높이려면 필연적으로 Precision이 하락한다.

- **F2**: Recall 0.375 → 0.411 (+0.036), 경보수 63 → 113 (1.8배). 오탐이 2배 가까이 늘지만 놓치는 기업은 줄어든다.
- **Recall≥70%**: Recall 0.571, 그러나 경보수 696개 — 정상 기업 640개를 오탐. 실운영 불가능.
- **Recall≥80%**: Recall 0.696, 경보수 1,026개. 실질적 의미 없음.

## 4. 결론

| 판단 | 내용 |
|------|------|
| 운영 권장 threshold | F1 전략 (threshold=0.097) — PR-AUC·Recall·Precision 균형 |
| Recall 추가 확보 필요 시 | F2 전략 (threshold=0.062) — Recall +0.036, 경보수 약 2배 허용 가능하면 선택 |
| Threshold 조정의 한계 | PR-AUC 0.2876이 모델의 천장. Recall을 의미있게 높이려면 모델 자체를 개선해야 한다 |

# 📌 exp_011: RF Optuna 하이퍼파라미터 튜닝

## 0. 실험 목적

exp_010에서 threshold 조정만으로는 PR-AUC 천장을 넘을 수 없음을 확인했다. RF 하이퍼파라미터를 Optuna로 탐색하여 모델 자체의 PR-AUC를 높일 수 있는지 확인한다.

## 1. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A / signed_log1p |
| Optuna CV | GroupKFold(5-fold, groups=stock_code), 목적함수: CV 평균 PR-AUC |
| Sampler | TPESampler(seed=42) |
| Trials | 50 |
| 탐색 공간 | n_estimators(100~600, step=50), max_depth(4~20), min_samples_leaf(1~20), max_features(sqrt/log2/0.3/0.5) |

## 2. 최적 하이퍼파라미터

CV 최고 PR-AUC: **0.1462** (baseline CV: 0.1190)

| 파라미터 | 최적값 | baseline |
|----------|--------|----------|
| n_estimators | 600 | 200 |
| max_depth | 14 | 10 |
| min_samples_leaf | 8 | 5 |
| max_features | 0.5 | sqrt (≈0.18) |

## 3. Test 성능 비교

| 모델 | Threshold | Valid PR-AUC | Test PR-AUC | Test Recall | Test Precision | Test F1 |
|------|-----------|-------------|-------------|-------------|----------------|---------|
| Baseline | 0.097 | 0.1190 | **0.2876** | **0.3750** | **0.3333** | **0.3529** |
| Tuned (F1 thr) | 0.155 | 0.1122 | 0.2496 | 0.2857 | 0.4324 | 0.3441 |
| Δ (Tuned − Base) | - | −0.0068 | **−0.0380** | −0.0893 | +0.0991 | −0.0088 |
| Tuned (F2 thr) | 0.059 | 0.1122 | 0.2496 | 0.4286 | 0.1446 | 0.2162 |

## 4. 해석

**Optuna 튜닝이 오히려 test PR-AUC를 낮췄다 (0.2876 → 0.2496, −0.038).**

두 가지 원인이 복합적으로 작용한다.

1. **CV-Test 괴리**: Optuna가 최적화한 CV PR-AUC는 train 내부 성능이다. Valid set 양성이 39개로 적어 valid 신호가 불안정하고, CV 점수와 test 점수 간 상관이 낮다.

2. **복잡한 모델의 과적합**: max_depth 14 + max_features 0.5 조합은 train에 더 잘 맞지만 test로 일반화가 덜 된다. baseline의 max_depth=10, max_features=sqrt(≈0.18)가 이 데이터에서는 더 적절한 정규화였다.

F2 threshold 적용 시 Recall 0.4286(+0.054)이나, Precision이 0.33→0.14로 절반 이하로 떨어져 실운영 부담이 크다.

## 5. 결론

| 판단 | 내용 |
|------|------|
| RF 하이퍼파라미터 튜닝 효과 | 없음. baseline이 더 우수 |
| RF 성능 천장 추정 | Test PR-AUC ~0.29 수준이 RF의 한계로 판단 |
| baseline 유지 | n_estimators=200, max_depth=10, min_samples_leaf=5, max_features=sqrt |

# 📌 exp_012: XGBoost Optuna 튜닝

## 0. 실험 목적

RF가 하이퍼파라미터 튜닝으로 개선되지 않자 모델 자체를 변경한다. XGBoost는 부스팅 기반으로 RF와 다른 귀납 편향을 가지며, `scale_pos_weight` 파라미터로 클래스 불균형을 직접 처리할 수 있다. RF 대비 우위 여부를 확인한다.

## 1. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A / signed_log1p |
| Optuna CV | GroupKFold(5-fold, groups=stock_code), 목적함수: CV 평균 PR-AUC |
| Sampler | TPESampler(seed=42) |
| Trials | 50 |
| scale_pos_weight 기준 | 자동 계산 neg/pos = **124.4** |
| 탐색 공간 | n_estimators, max_depth(3~10), learning_rate(log-scale), subsample, colsample_bytree, min_child_weight, gamma, reg_alpha, reg_lambda, scale_pos_weight(62~248) |

## 2. 최적 하이퍼파라미터

CV 최고 PR-AUC: **0.1295**

| 파라미터 | 최적값 | 비고 |
|----------|--------|------|
| n_estimators | 200 | - |
| max_depth | 3 | 얕은 트리 (강한 정규화) |
| learning_rate | 0.095 | - |
| subsample | 0.630 | - |
| colsample_bytree | 0.356 | - |
| min_child_weight | 17 | 매우 높음 (강한 정규화) |
| gamma | 2.74 | 높음 |
| reg_alpha | 2.27 | L1 정규화 강함 |
| reg_lambda | 3.18 | L2 정규화 강함 |
| scale_pos_weight | 104.3 | 기준값 124.4보다 낮음 |

## 3. Test 성능 비교

| 모델 | Test PR-AUC | Test Recall | Test Precision | Test F1 |
|------|-------------|-------------|----------------|---------|
| **RF Baseline (exp_008)** | **0.2876** | **0.3750** | **0.3333** | **0.3529** |
| XGB Baseline | 0.1861 | 0.2500 | 0.4375 | 0.3182 |
| XGB Tuned (F1 thr) | 0.2078 | 0.3036 | 0.1717 | 0.2194 |
| XGB Tuned (F2 thr) | 0.2078 | 0.3036 | 0.1717 | 0.2194 |

*F1/F2 threshold가 동일 (0.665): 두 전략이 같은 운영점을 선택*

## 4. 해석

**XGBoost는 RF보다 전반적으로 열위다.** 튜닝 후에도 Test PR-AUC 0.2078로 RF baseline(0.2876)에 미치지 못한다.

Optuna가 극도로 보수적인 파라미터를 선택한 것(max_depth=3, min_child_weight=17, gamma/alpha/lambda 모두 높음)은 XGBoost가 이 데이터에서 과적합에 취약하다는 신호다.

**RF가 XGBoost보다 강한 이유:**
- 124:1 극심한 불균형 환경에서 배깅 기반 RF의 앙상블 평균화가 부스팅보다 안정적
- scale_pos_weight가 그래디언트를 보정하지만, 초기 트리들이 다수 클래스에 쏠리는 부스팅 특성을 완전히 극복하지 못함
- RF의 per-tree 피처 샘플링(max_features=sqrt)이 자연스럽게 다양한 트리를 만들어 소수 클래스 신호를 보존

## 5. 결론

| 판단 | 내용 |
|------|------|
| 모델 선택 | RF 유지. XGBoost는 이 문제에서 열위 |
| XGBoost 재시도 조건 | 데이터 규모가 크게 늘거나 피처가 이질적으로 확장될 경우 재검토 가능 |

# 📌 exp_013: 피처 엔지니어링 실험 (missing flag / YoY / 재무복합지표)

## 0. 실험 목적

박민서(v3) 실험 분석에서 `_missing` flag 피처가 박민서 방식에서 강한 신호가 된다는 점을 확인했다. 동일 아이디어를 우리 fixed_N1 라벨링에 적용했을 때 효과가 있는지, 그 외 추가 피처(YoY 성장률 복원, 재무 복합 지표)의 기여도를 단계별로 측정한다.

## 1. 피처 구성

### 원본 데이터 결측 현황 (combined_raw.csv 기준)

| 구분 | 컬럼 | 결측률 | 처리 방식 |
|------|------|--------|-----------|
| 상각비 | 무형자산상각비, 유형자산상각비, 감가상각비 | 72~75% | CF=0 (0으로 채움) |
| 구조적 결측 | 비유동자산장기적합률 | 54% | 섹터·분기 중앙값 보간 |
| YoY 성장률 | 매출액증가율, 순이익증가율, 영업이익증가율 | 13~16% | 기존 실험에서 제외 |
| 기타 | 재고자산회전율, 매출원가율, 매출총이익률 | 7~11% | 섹터·분기 중앙값 보간 |

### 단계별 피처 추가

| Step | 피처 수 | 추가 내용 |
|------|---------|-----------|
| Step 0 — baseline | 33 | exp-A 현행 피처 |
| Step 1 — +missing | 43 | 결측 flag 10개 (결측 여부 → 0/1) |
| Step 2 — +yoy | 46 | 제외했던 YoY 성장률 3개 복원 (median 보간) |
| Step 3 — +distress | 49 | 재무 복합 지표 3개 추가 |

**재무 복합 지표 (Step 3):**
- `leverage_stress` = 부채비율 × 차입금의존도 / 100 (복합 레버리지 위험)
- `profit_vs_leverage` = 부채비율 / (총자본영업이익률 + 100) (수익성 대비 레버리지)
- `liquidity_composite` = 유동비율 + 현금비율 (복합 유동성)

## 2. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A / signed_log1p |
| Model | RF (baseline params, random_state=42) |
| Threshold | valid F1 최적화 |
| 결측 flag 소스 | combined_raw.csv (전처리 이전 원본) — processed CSV와 (stock_code, year, quarter) join |

## 3. 결과

| Step | 피처 수 | Valid PR-AUC | Test PR-AUC | Test Recall | Test Precision | Test F1 |
|------|---------|-------------|-------------|-------------|----------------|---------|
| Step0_baseline | 33 | 0.1190 | **0.2876** | **0.3750** | **0.3333** | **0.3529** |
| Step1_+missing | 43 | 0.1129 | 0.2458 | 0.3393 | 0.3455 | 0.3423 |
| Step2_+yoy | 46 | 0.1158 | 0.2704 | 0.3214 | 0.2812 | 0.3000 |
| Step3_+distress | 49 | 0.1152 | 0.2602 | 0.2857 | 0.3200 | 0.3019 |

## 4. 해석

**어떤 피처 추가도 baseline PR-AUC(0.2876)를 넘지 못했다.**

### missing flag가 박민서 방식에서는 효과적이지만 우리 방식에서는 효과 없는 이유

박민서의 라벨 정의(상폐 기업의 모든 연도 = label 1)에서는 상폐 기업이 수년에 걸쳐 보이는 결측 패턴이 신호가 된다. 예를 들어 결국 상폐되는 기업이 감가상각비를 수년 전부터 미신고하는 패턴이 label=1과 체계적으로 연결된다.

우리의 fixed_N1 라벨링에서는 "상폐 1년 전"에만 label=1이다. 그 1년 전 시점의 결측 패턴은 건강한 기업의 결측 패턴과 통계적으로 유사하다 — 예를 들어 재고가 없는 서비스 기업은 건강하더라도 재고자산회전율이 결측이다. 결측이 "곧 상폐할 것"과 직결되지 않기 때문에 flag 피처가 노이즈로 작용한다.

### YoY 성장률 복원 효과

Step1 대비 PR-AUC +0.025(0.2458 → 0.2704). 부분적 회복이 있지만 baseline을 넘지 못한다. 13~16% 결측을 중앙값으로 보간하면서 발생하는 노이즈가 정보 이득을 상쇄한다.

### 재무 복합 지표 효과 없음

Step2 대비 PR-AUC −0.010. RF는 이미 원본 피처의 곱/비율 조합을 트리 분할로 내부적으로 발견할 수 있어, 사전 정의된 복합 지표가 추가 정보를 제공하지 못한다.

## 5. 결론

| 피처 | 판단 |
|------|------|
| missing flag | 효과 없음. fixed_N1 라벨링에서 결측 패턴이 1년 전 상폐와 무관 |
| YoY 성장률 복원 | 부분 회복(+0.025) 있으나 baseline 미달, 보간 노이즈가 원인 |
| 재무 복합 지표 | 효과 없음. RF가 원본 피처로 이미 파악 가능 |
| 최종 피처셋 | exp-A 33개 피처 유지 |

# 📌 exp_014: RF class_weight 실험

## 0. 실험 목적

`class_weight='balanced'` / `'balanced_subsample'` 파라미터로 RF가 클래스 불균형을 내부적으로 보정할 수 있는지 확인한다. 단일 파라미터 변경만으로 PR-AUC를 높일 수 있으면 비용 대비 효과가 가장 높은 개선 방법이다.

## 1. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A / signed_log1p |
| Model | RF (baseline params) |
| 클래스 비율 | neg:pos = 124.4:1 |
| 비교 variant | None (baseline) / balanced / balanced_subsample |

- `balanced`: 전체 train 기준 클래스 빈도 역비율로 sample weight 부여
- `balanced_subsample`: 각 트리의 bootstrap 샘플 기준으로 가중치 재계산

## 2. 결과

| Variant | Threshold | Test PR-AUC | Test Recall | Test Precision | Test F1 |
|---------|-----------|-------------|-------------|----------------|---------|
| None (baseline) | 0.097 | **0.2876** | **0.3750** | **0.3333** | **0.3529** |
| balanced (F1 thr) | 0.721 | 0.1798 | 0.0714 | 0.4444 | 0.1231 |
| balanced (F2 thr) | 0.427 | 0.1798 | 0.3393 | 0.1301 | 0.1881 |
| bal_subsample (F1 thr) | 0.706 | 0.1837 | 0.0893 | 0.5000 | 0.1515 |
| bal_subsample (F2 thr) | 0.425 | 0.1837 | 0.3571 | 0.1408 | 0.2020 |

## 3. 해석

**balanced / balanced_subsample 모두 PR-AUC가 0.18 수준으로 급락했다.**

124:1 불균형에서 `balanced`는 소수 클래스에 124배 가중치를 부여한다. 이는 과보정이다. 다수 클래스 패턴 학습이 심각하게 방해받아 모델 분류 능력(ranking) 자체가 저하된다. threshold가 0.72까지 치솟는 것도 같은 이유다 — 예측 확률 분포 자체가 왜곡되어 유효한 threshold 구간이 사라진다.

F2 threshold로 낮추면 Recall은 회복(~0.35)되지만 PR-AUC 자체가 낮아 실질적 의미가 없다.

## 4. 결론

| 판단 | 내용 |
|------|------|
| class_weight 효과 | 없음. 오히려 악화 |

# 📌 exp_016: SMOTE 오버샘플링

## 0. 실험 목적

학습 데이터에서 소수 클래스를 합성 오버샘플링하여 불균형을 직접 해소한다. SMOTE와 그 변형(SMOTEENN, BorderlineSMOTE)이 fixed_N1 환경에서 PR-AUC를 높이는지 확인한다.

## 1. 실행 조건

| 항목 | 값 |
|------|-----|
| Dataset | fixed_N1 / exp-A / signed_log1p |
| Model | RF (baseline params) |
| SMOTE 적용 | train 데이터에만 적용 (valid/test 원본 유지) |
| SMOTE 적용 시점 | signed_log1p 변환 후 피처 공간에서 합성 |

|---------|------|-----------------|

| SMOTE_10 | sampling_strategy=0.10 (10:1 목표) | 3,521 |
| SMOTE_20 | sampling_strategy=0.20 (5:1 목표) | 7,042 |
| SMOTE_auto | sampling_strategy='auto' (1:1 목표) | 35,210 |
| SMOTEENN_10 | SMOTE + ENN 노이즈 제거, 0.10 | 2,218 |
| Borderline_10 | BorderlineSMOTE, 0.10 | 3,521 |

## 2. 결과

| Variant | N학습 | 합성 수 | Test PR-AUC | Test Recall | Test Precision | Test F1 | Δ vs baseline |
|---------|-------|---------|-------------|-------------|----------------|---------|---------------|
| None (baseline) | 35,493 | 0 | **0.2876** | **0.3750** | **0.3333** | **0.3529** | — |

*오버샘플링 비율이 클수록 PR-AUC 하락 폭이 커짐*

## 3. 해석

**모든 SMOTE 변형이 baseline을 넘지 못했다. SMOTEENN_10이 −0.0023으로 가장 근접하지만 사실상 동등 수준이고 오히려 Recall이 0.375→0.196으로 급락한다.**

SMOTE가 fixed_N1에서 효과 없는 구조적 이유:

1. **양성 클래스 클러스터 희박**: SMOTE는 k-최근접 양성 이웃 간 보간으로 합성 샘플을 만든다. fixed_N1 양성 샘플들(상폐 1년 전 기업)은 재무비율 공간에서 퍼져 있어 — 업종·규모·상폐 원인이 모두 다름 — 보간이 의미 없거나 노이즈 영역의 샘플을 생성한다.

2. **PR-AUC는 ranking 능력**: SMOTE는 decision boundary를 조정하는 것이지, 모델이 양성을 높은 확률로 **순위 매기는** 능력을 향상시키지 않는다. 합성 샘플이 많을수록 오히려 다수 클래스의 진짜 경계 정보가 희석된다.

3. **1년 전 신호 약도**: 상폐 1년 전 재무비율이 건강한 기업과 통계적으로 유사하다는 근본 문제는 오버샘플링으로 해결할 수 없다. 신호 자체가 약하다.

## 4. 결론

| 판단 | 내용 |
|------|------|
| SMOTE 효과 | 없음. 모든 variant baseline 열위 |
| SMOTEENN_10 | 가장 근접(−0.0023)이나 Recall 절반으로 하락 |
| 원인 | 양성 클러스터 희박 + 1년 전 신호 약도 + PR-AUC는 ranking 개선 필요 |

# 📌 exp_018: fixed_N2 레이블링 비교

## 0. 실험 목적

fixed_N1이 "상폐 1년 전"만 label=1로 보는 것과 달리, fixed_N2는 "상폐 정확히 2년 전"을 label=1로 정의한다. 2년 전 시점에서 재무 악화 신호가 더 뚜렷하게 나타나는지, PR-AUC가 개선되는지 확인한다.

## 1. 레이블링 차이

| 구분 | label=1 정의 | Train 양성 | Valid 양성 | Test 양성 |
|------|-------------|-----------|-----------|---------|
| fixed_N1 | 상폐연도 **1년 전** 전 분기 | 283 | 39 | **56** |
| fixed_N2 | 상폐연도 **2년 전** 전 분기 | 293 | 60 | **24** |
| fixed_N3 | 상폐연도 **3년 전** 전 분기 | 316 | 24 | **0** |

- N2 valid 양성이 39→60으로 증가: 2025년 상폐 예정 기업의 2023년 데이터
- N2 test 양성이 56→24로 감소: 2026년 상폐 예정 기업의 2024년 데이터
- N3 test 양성 = 0: 2027년 상폐 예정 기업은 2024년 데이터가 아직 학습/평가 불가

## 2. 결과

| N | Threshold | Valid PR-AUC | Valid pos | Test PR-AUC | Test Recall | Test Precision | Test F1 | Test pos |
|---|-----------|-------------|-----------|-------------|-------------|----------------|---------|---------|
| N1 (F1) | 0.097 | 0.1190 | 39 | **0.2876** | **0.3750** | **0.3333** | **0.3529** | 56 |
| N1 (F2) | 0.062 | 0.1190 | 39 | 0.2876 | 0.4107 | 0.2035 | 0.2722 | 56 |
| N2 (F1) | 0.049 | **0.1540** | 60 | 0.0100 | 0.1250 | 0.0135 | 0.0244 | 24 |

## 3. 해석

**Valid PR-AUC는 N2(0.1540)가 N1(0.1190)보다 높다 — 2년 전 재무 데이터에 실제로 더 강한 구분 신호가 존재한다.** 그러나 Test PR-AUC는 N2에서 0.0100으로 붕괴한다.

붕괴 원인의 구조적 이해:

1. **Test positive 정체**: N2 test positive = "2026년에 상폐될 기업의 2024년 데이터". 이 기업들은 2024년 현재 아직 2년의 여유가 있어 재무제표가 정상처럼 보일 가능성이 높다.

2. **분포 이동**: 학습 데이터(2015~2022)의 N2 양성 패턴과 test(2024)의 N2 양성 패턴 사이에 시장 환경 변화로 인한 분포 차이가 있을 수 있다.

3. **소표본 불안정**: 24개는 신뢰할 수 있는 PR-AUC 추정에 부족하다. 그러나 precision=0.0135(모델이 예측한 양성의 1.35%만 실제 양성)는 단순 노이즈로 설명되지 않는 모델 실패를 보여준다.

fixed_N3는 test positive=0으로 평가 자체가 불가능하다.

## 4. 결론

| 판단 | 내용 |
|------|------|
| N2 valid 신호 | N1보다 강함 (0.154 vs 0.119) — 2년 전 재무악화 신호는 존재 |
| N2 test 일반화 | 실패. 24개 양성, 분포 이동, 미래 상폐 기업 패턴 예측 어려움 |
| N3 | test 양성 0개. 평가 불가 |
| 결론 | 현재 시계열 범위(2015~2024) 내에서 fixed_N2/N3는 test 평가가 구조적으로 불리 |

## 핵심 결론: 현재 데이터의 신호 한계

9개 실험(exp_010~018)을 통해 **모든 방향의 개선 시도가 RF baseline PR-AUC 0.2876을 넘지 못했다**. 이는 모델·피처·샘플링의 문제가 아니라 **fixed_N1 라벨링 + 재무비율 스냅샷이라는 데이터 자체의 한계**다.

상폐 1년 전 재무비율은 많은 경우 아직 정상 기업과 통계적으로 구분되지 않는다. 일부 기업은 재무 악화가 빠르게 진행되어 1년 전 데이터에 신호가 있지만, 다른 기업은 시장·규제·오너 리스크 등 재무제표에 나타나지 않는 이유로 상폐된다.

## PR-AUC 0.29의 의미

PR-AUC 0.2876은 랜덤(≈ 클래스 비율 = 0.008)보다 약 36배 높은 수치다. 완전한 무작위 예측이 아니라 재무비율에서 실질적인 신호를 찾고 있다는 의미이지만, 실운영 조기경보 시스템으로서의 정밀도는 부족하다.

## 추가 개선을 위해 필요한 것

현재 데이터 범위 내의 개선은 한계에 도달했다. 의미 있는 성능 향상을 위해서는 다음이 필요하다.

| 방향 | 내용 | 기대 효과 |
|------|------|----------|
| 비재무 데이터 추가 | 감사의견, 최대주주 변경, 거래량 이상, 공시 패턴 | 재무제표에 미포착되는 상폐 신호 포착 |
| 시계열 피처 | 전 비율의 다년간 추세(기울기, 가속도) | 악화 속도 자체를 신호로 활용 |
| 시장 데이터 | 시가총액, 주가 변동성, 공매도 비율 | 시장 선행 신호 반영 |
| 더 많은 학습 데이터 | KOSPI 포함 확장, 더 긴 역사 | 양성 샘플 증가로 모델 안정화 |
