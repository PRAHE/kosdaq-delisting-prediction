# 아웃라인 (1)

### 1. 문제 정의 & 라벨링 설계

- 상장폐지 중 재무/회계 기반 문제성 상폐만 라벨에 포함(합병/자진상폐 등 제외)
- 예측 시점 정의: 기준시점 T(분기/연 단위), 예측 H개월(예:12)
- 라벨 정의: y{T)=1 if T 이후 H개월 내 문제성 상폐 발생
- 조기경보(Recall/F2 중심)

⇒ 산출물:

- 라벨링 규정서(포함/제외 사유 표)
- 실험 설정 문서(T 단위, H, K(rolling window 길이)

### 2. 데이터 수집 & 통합

상폐 목록 + 기업 식별 + 재무/공시 를 한 테이블로 합치기

- KRX/KIND: 상폐 기업 목록(상폐일/사유/종목코드)
- DART: 재무제표(분기/연), 공시 메타
- 기업 식별키 정리: 종목코드/기업고유번호(corp_code), 기간 키(분기)

⇒ 산출물:

- Raw 수집 데이터(파일/DB)
- 기업-기간 단위 마스터 테이블 스키마 설계(컬럼 정의서)

### 3. Point-in-Time + Rolling Winow 데이터셋 구성

미래정보 누수 없이, T 시점에 알 수 있던 정보로 feature 구성

- Point-in-Time 필터: 공시/제출일 $\le$ T 만 사용
- Rolling window 샘플: 각 (기업, T)에 대해 과거 K분기 데이터를 feature로 구성
- 샘플 생성 로직 고정

⇒ 산출물: 

- 모델 입력용 학습 데이터셋 (기업-시점 단위)

### 4. 전처리/결측/파생변수

모델이 학습 가능한 feature를 안정적으로 만들기

- 결측: 산업군 중앙값 + missing indicator(기본) / Iterative(옵션)
- 파생변수 생성
- 이상치 처리(winsorize/robust scaling 등) - 비율 피처 유의
- fit은 train에서만, valid/test는 transform만

⇒ 산출물:

- 전처리 파이프라인 코드
- feature dictionary(각 변수 의미/계산식)

### 5. 검증 설계 (시간 + 기업 누수)

실전 성능 측정

- 기본: Time split (train: 과거 / valid / test: 미래)
- 추가: Group split(CV) - 기업이 fold를 넘지 않게(기업 암기 방지)
- 최종: 기업 holdout test - 튜닝에 전혀 쓰지 않는 기업 세트로 마지막 확인

⇒ 산출물:

- 분할 규칙 문서 + 분할된 인덱스 저장
- 실험 로그 템플릿

### 6. 베이스라인(Global 모델)

군집/듀얼트랙 없이도 작동하는 강한 baseline 만들기

- 모델: XGBoost/LightGBM/CatBoost 중 1~2개
- 불균형: scale_pos_weight/sample weight
- 평가: PR-AUC, Recall, F2, Calibration(Brier/curve)

⇒ 산출물:

- Baseline 성능표(모든 지표)
- 주요 실패 케이스 분석

### 7. 산업/군집 반영 확장

복잡도를 계단식으로 올려서 실패 확률 줄이기

1. 단일 모델 + industry/cluster_id feature
2. 산업별 모델(가능하면) or 군집별 모델
3. 듀얼트랙(Global + Cluster) + 가중 결합
- 1차 군집: KSIC/KRX 산업코드(도메인 기반)
- 2차 군집(옵션): 재무비율 벡터 + cosine + 계층군집(안정성 체크 포함)
- 듀얼트랙 결합: $P_{final} = w_1*P_{global}+w_2*P_{cluster}$
- $w_2$규칙: 샘플 수 적으면 가중치 적게 + 가능하면 valid에서 튜닝

⇒ 산출물:

- Baseline vs + IndustryFeature vs DualTrack 비교 실험표
- 군집 품질/안정성 리포트(실루엣, 분포, 시간 안정)

### 8. 확률 보정(Calibration)

0~100 점수가 진짜 확률처럼 해석되게

- Platt(표본 적을 때) vs Isotonic(표본 충분할 때) 비교
- 듀얼 결합을 쓴다면 결합 후 최종 출력에 대해 1회 더 보정 옵션 고려

⇒ 산출물:

- Calibration curve, Brier score 비교표
- 보정 전/후 점수 해석 예시

### 9. 리스크 점수/임계치/조사 우선순위 로직

모델을 “조기경보 시스템”으로 변환

- 점수화: `RiskScore = 100 * P_calibrated`
- 임계치: valid에서 threshold sweep → Recall/F2 최대
- Top-N: 상위 1~3% 모니터링 같은 정책 병행(옵션)
- 조사 우선순위:
    1. RiskScore
    2. SHAP 주요 원인(Reason code)
    3. (옵션) 데이터 신뢰도/군집 신뢰도(flag)

⇒ 산출물

- 운영 규칙 문서(“언제 경보?”, “Top-N은 몇 개?”)
- 샘플 리포트(기업 1개에 대한 점수+SHAP 설명 페이지)

    - 기업 상세 페이지: 점수 추이 + 주요 피처 추이 + SHAP Top5

    5. 실험/결과(비교 실험, ablation)
    6. 대시보드/사례 분석
    7. 결론/한계/향후과제(운영화 확장 포함)
