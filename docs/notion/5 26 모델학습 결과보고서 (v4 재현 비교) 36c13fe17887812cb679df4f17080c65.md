## 0. 실험 목적

v4 보고서는 `delisted/` 폴더 모든 분기 행을 양성(=1)으로 두는 단순 라벨링을 쓴다. 기존 파이프라인(fixed_N1)은 상폐 시점 정확히 1년 전 분기만 양성으로 보는 더 엄격한 라벨링이다. 두 방식의 성능 차이가 **모델**의 차이인지 **라벨 정의**의 차이인지 분리하기 위해, **동일한 raw 데이터·split 정책·XGBoost 하이퍼파라미터** 위에서 라벨링만 바꿔 비교했다.

*💡 두 variant는 raw / feature / split / 모델 / 하이퍼파라미터가 100% 같고, 양성 정의만 다르다.*

| **Variant** | **양성 정의** |
| --- | --- |
| `all_delisted` | PDF v4 방식. delisted/ 폴더 모든 행 = 1 (상폐 시점 구분 없이 전부 양성) |
| `only_n1` | fixed_N1 방식. 상폐 연도 정확히 1년 전 분기만 = 1, 그 외 상폐기업 행은 제거 |

## 1. 실행 조건

| **항목** | **값** |
| --- | --- |
| 입력 raw | `data/raw/raw/{healthy,delisted}/<sector>/*.json (56,235개)` |
| 보고기간 | Q1 / H1 / Q3 / ANNUAL 4종 모두 |
| Split | Group Split (train: 70%, validation: 15%, test: 15%) — PDF v4와 동일 |
| 성장률 | 동일 quarter year-1 YoY (매출액 / 순이익 / 영업이익) |
| 결측 처리 | (sector×quarter) 중앙값 → quarter 중앙값 → global → 0, _missing flag 동반 |
| 거시경제 | macro_quarterly.csv 6개 컬럼 left join (year, quarter 키) |
| Feature 수 | 70 (재무비율 30 + missing flag 30 + period flag 4 + macro 6) |
| 모델 | XGBoost (PDF grid1_02 그대로: max_depth=3, lr=0.03, n_estimators=400, subsample=0.8, colsample_bytree=0.8, scale_pos_weight=neg/pos) |
| Threshold | validation F1 최대화 grid (0.05–0.95, step 0.01) |
| Seed | 42 |
- seed : 랜덤분할을 고정한다.
    - train/validation/test에 들어가는 종목코드 분할 고정
- 동일 quarter year-1 YoY (매출액 / 순이익 / 영업이익)
    - 같은 분기를 작년 같은 분기와 비교해서 성장률을 계산
    - 계절성을 제거하고 진짜 성장/악화를 보기 위함

## 4. 분석

### 4.1 라벨 정의로 인한 점수 차이

- **양성 수가 1/7로 감소 (2,575 → 378)**
- **불균형 7배 악화 (20:1 → 148:1)**

이 패턴이 말하는 것:

- **모델의 ranking 능력은 only_n1에서 더 좋다** (ROC-AUC ↑, recall 유지).
    - 즉 “정확히 상폐 1년 전” 시점은 신호가 더 강하다.
- 그러나 양성이 너무 적어 운영점 F1이 폭락한다
    - precision이 무너지면서 false positive 비용이 커진다.
- recall이 두 variant에서 거의 같은 이유는 **두 라벨링 모두 “상폐 1년 전 시점” 행을 핵심 신호로 학습**하기 때문.
    - all_delisted는 추가로 상폐 2~5년 전 행도 양성으로 보지만 이는 noise성 양성으로 볼 수 있다.
    - precision은 올라가고 PR-AUC ranking이 살아남는다.

- scale_pos_weight를 잡으면 sigmoid 출력이 전체적으로 위로 끌려올라서 best threshold도 자연스레 높아진다.
