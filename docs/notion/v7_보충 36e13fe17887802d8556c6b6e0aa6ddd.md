# **XGBoost 추가 실험 비교 보고서**

## **1. 실험 목적**

이번 추가 실험의 목적은 상폐 예측 모델의 F1 score를 조금 더 개선할 수 있는지 확인하는 것이다. 이전 실험에서는 상폐일 전 365일 기준과 730일 기준을 비교했고, 730일 기준에서 test F1이 더 안정적으로 나왔다. 이번에는 그 중간값인 540일 라벨 실험을 추가하고, 730일 기준에서는 threshold 전략과 `scale_pos_weight` 범위를 더 넓혀 다시 학습하였다.

이번 실험에서 확인하려고 한 핵심은 다음과 같다.

- 365일과 730일 사이의 중간값인 540일 기준이 더 좋은 균형점이 될 수 있는지 확인
- 730일 기준에서 `scale_pos_weight`를 2.5배, 3.0배까지 늘렸을 때 성능이 개선되는지 확인
- threshold 전략을 더 세분화했을 때 F1 score가 개선되는지 확인
- feature 제거는 `all_features`, 중요도 하위 5개 제거, 중요도 하위 10개 제거까지만 비교

## **2. 실험 설정**

이번 실험에서는 기존 full raw feature CSV를 그대로 사용하고, label만 다시 정의하였다. 즉, 재무비율 계산 방식은 기존과 같고, 상폐일까지 남은 기간 기준만 바꾸었다.

| 실험 | label=1 기준 | 설명 |
| --- | --- | --- |
| 365일 기준 | 상폐일 전 365일 이내 | 상폐 직전 1년 예측 |
| 540일 기준 | 상폐일 전 540일 이내 | 365일과 730일 사이의 중간 실험 |
| 730일 기준 | 상폐일 전 730일 이내 | 상폐 전 약 2년 이내 위험 예측 |

이번에 새로 실행한 실험은 `20260528_0040_relabel540_tuned`, `20260528_0040_relabel730_tuned`이다.

## **3. 라벨 분포 비교**

| 기준 | 전체 행 수 | label=1 행 수 | validation positive | test positive |
| --- | --- | --- | --- | --- |
| 365일 | 55,072 | 275 | 40 | 43 |
| 540일 | 55,870 | 545 | 77 | 85 |
| 730일 | 55,870 | 740 | 100 | 116 |

365일 기준에서는 label=1이 275개로 매우 적었다. 540일 기준에서는 545개, 730일 기준에서는 740개로 늘어났다. label=1 수가 늘어나면서 validation/test set에도 positive 샘플이 더 많이 들어갔고, 모델이 상폐 위험 패턴을 학습할 기회도 늘어났다.

## **5. 730일 추가 튜닝 결과**

730일 기준 추가 튜닝에서는 `scale_pos_weight`를 2.5배, 3.0배까지 추가하고 threshold 전략도 더 세분화하였다. 그러나 validation F1 기준 best model은 기존 730일 실험과 동일하게 `feature_drop_low_importance_10`이었다.

| 항목 | 값 |
| --- | --- |
| best experiment | feature_drop_low_importance_10 |
| feature set | drop_low_importance_10 |
| threshold strategy | f1_max |
| threshold | 0.6750 |
| validation F1 | 0.2092 |
| validation PR-AUC | 0.0962 |
| test precision | 0.2202 |
| test recall | 0.4138 |
| test F1 | 0.2874 |
| test PR-AUC | 0.2261 |

추가한 `scale_pos_weight` 2.5배, 3.0배 설정은 metrics에는 정상적으로 포함되었지만, validation F1 기준 best setting을 바꾸지는 못했다. 즉, 730일 기준에서는 기존 best 설정이 여전히 가장 안정적이었다.

## **6. 전체 결과 비교**

| 기준 | best model | validation F1 | validation PR-AUC | test precision | test recall | test F1 | test PR-AUC |

365일 기준은 validation F1은 가장 높았지만 test recall이 0.0233으로 매우 낮았다. 이는 test set에서 실제 상폐 위험 구간을 거의 잡지 못했다는 의미이다. 반면 730일 기준은 validation F1은 365일보다 낮았지만 test recall과 test F1이 훨씬 높았다.

따라서 이번 실험에서는 730일 기준이 가장 좋은 결과로 해석된다. 특히 test F1과 test PR-AUC가 가장 높고, positive 예측도 지나치게 적지 않았다.

## **8. 해석**

이번 실험에서 가장 중요한 변화는 label=1 범위를 넓힐수록 test set에서 상폐 위험 구간을 더 잘 잡기 시작했다는 점이다. 365일 기준은 상폐 직전성은 가장 강하지만 positive 샘플이 너무 적어서 모델이 거의 양성 예측을 하지 못했다. 540일 기준은 이 문제를 일부 완화했지만, 730일 기준만큼 안정적인 성능은 나오지 않았다.

730일 기준은 상폐 직전 1년보다 범위가 넓기 때문에 "상폐 직전 예측"이라기보다는 "상폐 전 약 2년 이내 위험 예측"에 가깝다. 하지만 현재 데이터 규모에서는 이 기준이 모델 학습에 더 적합했고, test F1도 가장 높았다.

다만 validation F1과 test F1의 차이가 있으므로, 이 결과를 과도하게 일반화하면 안 된다. 다음 단계에서는 730일 기준을 유지하되, threshold와 class weight를 더 세밀하게 조정하거나, 최종 모델 선택 기준에 validation PR-AUC를 더 적극적으로 반영하는 방식이 필요하다.

## **9. 결론**

이번 추가 실험의 결론은 다음과 같다.

- 540일 기준은 365일 기준보다 test F1이 개선되었지만, 730일 기준보다 낮았다.
- 730일 추가 튜닝은 기존 730일 best setting을 바꾸지는 못했다.
- 현재까지는 730일 기준 `feature_drop_low_importance_10` 모델이 가장 안정적인 결과이다.
- 최종 성능은 test F1 0.2874, test PR-AUC 0.2261이다.

따라서 현재 단계에서 가장 설명력 있는 기준은 `상폐일 전 730일 이내 = label 1`이다. 이 기준은 label=1 수를 충분히 확보하면서도 상폐 위험 예측이라는 목적을 유지할 수 있는 타협점으로 볼 수 있다.
