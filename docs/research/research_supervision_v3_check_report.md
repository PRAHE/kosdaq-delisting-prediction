## #7 — A2 관리종목 coverage 감사

### 질문
A2는 "coverage 11.8%(1,536 중 181 corp)"와 함께 **"관리종목은 감사의견(A1)과
중복, 한계효용 0"** 이라는 negative result를 냈다. 그러나 코스닥 상폐기업의
80%+가 관리종목을 경험한다는 통설과 괴리가 크다. 이 낮은 coverage가
**(a) 키워드 매칭 실패 / (b) corp_code 미매핑 / (c) 진짜 공시 부재** 중 무엇인지,
누락된 상폐기업의 **원본 거래소공시(`pblntf_ty='I'`)를 무필터 재수집**해 규명한다.

### 방법
- 패널의 상폐기업(delist_year 존재) 107개 추출.
- 캐시에 이벤트가 있는(covered) / 없는(uncovered) 기업 분리.
- uncovered 상폐기업의 'I' 공시를 무필터 재조회 → 기존 키워드(관리종목/투자주의/
  투자경고/단기과열)와 **확장 키워드**(상장폐지/실질심사/상장적격성/매매거래정지/
  정리매매) 둘 다로 재분류.

### 결과

| 항목 | 값 |
|------|-----|
| 상폐기업 | 107 |
| 기존 coverage (이벤트 보유) | 70 (**65.4%**) |
| uncovered (매핑됨) | 37 / unmapped 0 |
| **확장 키워드로 복구** | **+37 → 전부** |
| **복구 후 coverage** | **107 (100.0%)** |
| 진짜 부실공시 부재 | **0** |

기존 키워드가 놓친 report_nm Top:

| 건수 | report_nm |
|------|-----------|
| 40 | 기타시장안내(상장폐지 관련 이의신청서 접수) |
| 38 | 주권매매거래정지기간변경(개선기간 부여) |
| 32 | 주권매매거래정지기간변경(상장폐지 사유 발생) |
| 26 | 주권매매거래정지해제(상장폐지에 따른 정리매매 개시) |
| 20 | 기타시장안내(상장적격성 실질심사 사유추가 안내) |
| 17 | 주권매매거래정지기간변경(상장적격성 실질심사 대상 결정) |
| 16 | 기타시장안내(상장적격성 실질심사 대상 결정) |
| 15 | 주권매매거래정지(불성실공시법인 지정) |

### 결론
- **A2의 데이터 수집이 불완전했다.** 기존 키워드는 "관리종목"이라는 글자에
  의존했는데, 상폐 파이프라인의 **가장 직접적 공시(상장적격성 실질심사, 매매
  거래정지, 상장폐지 사유 발생, 정리매매)** 는 그 글자가 없어 전부 누락됐다.
  상폐기업 실제 coverage는 ~12%/65%가 아니라 **100%** 다.
- 따라서 **A2의 "관리종목은 중복·한계효용 0" negative result는 confounded**다 —
  가장 강한 상폐 신호를 제외한 채 만든 피처로 평가했기 때문. **재수집 후
  재평가가 필요**하다.
- ⚠ **단, 누수 주의.** 복구된 공시 중 다수(정리매매 개시, 상폐 사유 발생,
  매매거래정지 해제)는 상폐 **직전·당시** 발생이라 fixed_N1(상폐 1년 전) 라벨
  기준 누수 위험이 크다. 재평가 시 **엄격한 시점 컷오프**로 *조기* 공시(실질심사
  대상 결정, 개선기간 부여, 불성실공시 지정)만 사용하고, 말기 공시(정리매매 등)는
  배제해야 한다. 그래야 "coverage 향상"이 "누수"로 변질되지 않는다.

## #9 — v3(L7) "valid 0.42 착시" 재현

### 질문
SR §4.4는 v3의 "valid PR-AUC 0.42 → test 0.234 붕괴"를 **평가 비일관성**(valid를
v3 라벨로, test를 fixed_N1로 평가)에 의한 착시로 *추정*했으나 원본 0.42를
재현하진 못했다. 이를 직접 재현해 루프를 닫는다.

### 방법
- train = L7(v3, 상폐기업 모든 연도=1) 라벨로 RF(스냅샷) 5-seed 학습.
- 같은 valid 예측 확률을 **두 정답**으로 평가: (a) L1 fixed_N1 (공정), (b) L7 v3.

### 결과

| split | 정답 | 양성 (base rate) | PR-AUC |
|-------|------|------------------|--------|
| valid | L1 fixed_N1 | 39 (0.0077) | **0.193** |
| valid | L7 v3 | 123 (0.0244) | **0.359** |
| test | L1 fixed_N1 | 56 (0.0105) | 0.250 |
| test | L7 v3 | 80 (0.0151) | 0.208 |

### 결론
- **동일 모델·동일 예측인데 valid 정답만 L1→L7로 바꾸면 PR-AUC 0.193 → 0.359
  (+0.166)** — base rate가 3.2배(39→123) 오른 결과다. SR §4.4의 "평가 비일관성
  착시" 가설이 **메커니즘으로 확인**된다. 원본의 "0.42"는 valid를 v3 라벨로
  평가한 부풀려진 수치였다(정확한 0.42 vs 본 재현 0.36 차이는 원본의 모델·피처
  차이; 방향·규모는 일치).
- test에서는 오히려 하락(0.250→0.208) — base rate 증가가 작고(1.4배) 추가된
  양성이 더 어렵기 때문. **착시는 valid 특이적**이며 "라벨 확장이 항상 PR-AUC를
  올린다"는 단순 논리가 아님을 보여준다. SR의 "라벨링은 천장의 주요 원인이
  아니다" 결론을 보강한다.
