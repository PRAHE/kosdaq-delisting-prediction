## A. 추가 데이터 수집

### A1. 감사의견 (Audit Opinion)

**설명**  
연간 감사보고서의 감사의견 종류: 적정(Clean) / 한정(Qualified) / 부적정(Adverse) / 의견거절(Disclaimer).  
비적정 의견은 코스닥 상장규정상 2년 연속 발생 시 상폐 사유가 된다.

**이론적 근거**  
- Geiger & Raghunandan (2002): *Going-Concern Opinions and the Prediction of Corporate Failure.* — 비적정 의견이 향후 1~2년 내 기업 실패를 예측하는 강력한 신호임을 실증
- Lennox (1999): *Identifying Failing Companies: A Reevaluation of the Logit, Probit, and DA Approaches.* — 회계 변수에 감사의견을 추가하면 부실 예측 정확도가 유의미하게 향상
- KRX 코스닥 상장규정 제38조: "감사인의 감사의견이 2년 연속 부적정 또는 의견거절인 경우" 상폐 사유로 명시

**피처화 방법**
```
audit_opinion_t    : 당해연도 감사의견 (0=적정, 1=한정, 2=부적정/거절)
audit_opinion_t1   : 전년도 감사의견
audit_consecutive  : 비적정 연속 횟수 (0, 1, 2+)
going_concern_flag : 계속기업 불확실성 주석 여부 (0/1)
```

**수집 방법**  
DART OpenAPI `fnlttSinglAcnt` → `감사보고서` 문서에서 의견 텍스트 파싱.  
기존 DART 수집 파이프라인 확장으로 구현 가능.

**기대 효과**: ★★★★★ — 상폐 직전 기업의 비적정 의견 비율이 매우 높음. 단일 피처로 가장 강한 선행 신호  
**수집 난이도**: ★★☆☆☆

## B. 피처 엔지니어링 확장

### B1. 다년간 시계열 피처 (Lag Features)

**설명**  
현재 모델은 단일 연도 재무비율 스냅샷만 사용한다. t-1, t-2년 비율을 피처로 추가하면 추세와 속도 정보를 모델에 제공할 수 있다.

**이론적 근거**  
- Tinoco & Wilson (2013): *Financial Distress and Bankruptcy Prediction among Listed Companies.* — 다년간 회계 추세 변수가 단일 스냅샷보다 부실 예측력이 유의미하게 높음
- Beaver (1966): *Financial Ratios as Predictors of Failure.* JAR — 5년간 추세가 부실 징후의 핵심 신호임을 실증. 재무비율의 시계열 패턴 연구의 시초

**구현 방법**  
combined_raw.csv에 모든 연도 데이터가 있으므로 stock_code 기준 shift 연산으로 구현 가능.

```python
# 예시: 4분기 전(전년 동분기) 피처 생성
for col in RATIO_COLS:
    raw[f"{col}_lag1"] = raw.groupby("stock_code")[col].shift(4)   # 1년 전
    raw[f"{col}_lag2"] = raw.groupby("stock_code")[col].shift(8)   # 2년 전
    raw[f"{col}_delta1"] = raw[col] - raw[f"{col}_lag1"]           # 전년 대비 변화
    raw[f"{col}_accel"]  = raw[f"{col}_delta1"] - (raw.groupby("stock_code")[f"{col}_delta1"].shift(4))  # 변화의 변화(가속도)
```

추가 피처 수: 비율 27개 × (lag1, lag2, delta1, accel) = 최대 108개 추가.  
결측 처리 주의: lag 피처는 데이터가 충분히 긴 기업에만 유효.

**기대 효과**: ★★★★☆ — 현재 데이터로 즉시 구현 가능하며, 추세 정보 자체가 스냅샷보다 강한 신호일 가능성 높음  
**구현 난이도**: ★★☆☆☆

### B2. 섹터 상대 위치 피처 (Sector-Normalized)

**설명**  
같은 절대값의 부채비율이라도 섹터마다 정상 범위가 다르다. 각 비율을 섹터 내 상대적 위치로 정규화하면 섹터 이상치를 포착할 수 있다.

**이론적 근거**  
- Altman (1993): *Corporate Financial Distress and Bankruptcy.* — 산업 조정 Z-score(industry-adjusted)가 원본보다 예측력이 높음을 제안
- Ohlson (1980): *Financial Ratios and the Probabilistic Prediction of Bankruptcy.* — 산업별 차이가 부실 예측의 중요한 변수

**구현 방법**
```python
for col in RATIO_COLS:
    sector_median = df.groupby(["gics_sector", "quarter"])[col].transform("median")
    sector_iqr    = df.groupby(["gics_sector", "quarter"])[col].transform(lambda x: x.quantile(0.75) - x.quantile(0.25))
    df[f"{col}_sector_z"] = (df[col] - sector_median) / (sector_iqr + 1e-8)
    df[f"{col}_sector_rank"] = df.groupby(["gics_sector", "quarter"])[col].rank(pct=True)  # 섹터 내 백분위
```

**기대 효과**: ★★★☆☆ — 섹터 특성이 강한 재무비율(부채비율, ROE 등)에서 추가 신호 기대  
**구현 난이도**: ★★☆☆☆

### B3. Altman Z-score

**설명**  
1968년 Altman이 제안한 선형결합 부도 예측 지수. 5개 재무비율의 가중합으로 계산된다. 50년 이상 검증된 기업부실 예측의 표준 지표.

**이론적 근거**  
- Altman (1968): *Financial Ratios, Discriminant Analysis, and the Prediction of Corporate Bankruptcy.* JF — 5개 비율의 선형결합이 2년 내 파산을 68% 정확도로 예측
- Altman, Haldeman & Narayanan (1977): ZETA 모델 — Z-score 개선판
- Altman & Sabato (2007): *Modelling Credit Risk for SMEs.* — Z-score가 중소기업에서도 유효

**수식**  
원래 수식 (상장기업 기준):
```
Z = 1.2×X1 + 1.4×X2 + 3.3×X3 + 0.6×X4 + 1.0×X5

X1 = 운전자본 / 총자산           ≈ 순운전자본비율 / 100
X2 = 이익잉여금 / 총자산          ≈ 유보액/납입자본비율의 변형
X3 = EBIT / 총자산               ≈ 총자본영업이익률 / 100
X4 = 시가총액 / 총부채장부가치     → 시장 데이터 필요 (없으면 자기자본비율로 대리)
X5 = 매출액 / 총자산             ≈ 총자본회전율 / 100
```

X4에 시장 데이터가 없으면 Altman(1983)의 비상장 기업 수정 모형(X4를 장부 자기자본으로 대체) 사용 가능.

**기대 효과**: ★★★☆☆ — 이론적으로 검증된 지표이나 RF가 원본 비율로 유사한 조합을 이미 학습할 가능성이 있음  
**구현 난이도**: ★★☆☆☆ (시장 데이터 없이 근사 버전)

df["debt_x_credit_spread"]  = df["부채비율"] * df["credit_spread"]

## C. 모델 아키텍처 변경

### C1. CatBoost

**설명**  
Yandex가 개발한 경사부스팅 모델. 범주형 피처(gics_sector)를 원-핫 없이 직접 처리하고, Ordered Boosting으로 타겟 리케이지를 방지한다. XGBoost/LightGBM보다 소표본에서 안정적인 경향이 있다.

**이론적 근거**  
- Prokhorenkova et al. (2018): *CatBoost: Unbiased Boosting with Categorical Features.* NeurIPS — Ordered Boosting이 기존 부스팅의 예측 편향을 줄이고 소표본에서 과적합을 억제
- Hancock & Khoshgoftaar (2020): *CatBoost for Big Data: An Interdisciplinary Review.* — 금융 불균형 데이터에서 XGB 대비 우위 사례 다수 수록

**주요 하이퍼파라미터**
```python
CatBoostClassifier(
    iterations=500,
    depth=6,
    learning_rate=0.05,
    scale_pos_weight=124,   # 불균형 처리
    cat_features=["gics_sector"],  # 범주형 직접 처리
    eval_metric="PRAUC",
    random_seed=42,
)
```

**기대 효과**: ★★★☆☆ — XGB/LGB보다 소표본 성능이 안정적이라는 근거 있음. gics_sector를 범주형으로 직접 처리하는 것이 차별점  
**구현 난이도**: ★★☆☆☆

### C2. 불균형 전용 앙상블: EasyEnsemble / BalancedBagging

**설명**  
SMOTE가 합성 샘플 생성으로 실패한 것과 달리, 다수 클래스를 언더샘플링하여 균형 잡힌 서브셋으로 여러 분류기를 학습하는 앙상블 방법이다.

**이론적 근거**  
- Liu, Wu & Zhou (2009): *Exploratory Undersampling for Class-Imbalance Learning.* IEEE Trans. SMC — EasyEnsemble이 SMOTE보다 극심한 불균형(>100:1)에서 안정적임을 실증
- Wallace & Dahabreh (2012): 불균형 데이터에서 언더샘플링+앙상블이 오버샘플링보다 일반적으로 더 나은 AUC를 보임

**구현 방법**
```python
from imblearn.ensemble import EasyEnsembleClassifier, BalancedBaggingClassifier

eec = EasyEnsembleClassifier(n_estimators=50, random_state=42, n_jobs=-1)
# 내부적으로 다수 클래스를 RUS(Random Undersampling)하여 50개 AdaBoost 학습
```

SMOTE와의 차이: 합성 샘플이 아닌 원본 다수 클래스의 서로 다른 부분집합을 사용 → 정보 손실 없이 균형 달성.

**기대 효과**: ★★★☆☆ — SMOTE 실패 이후 언더샘플링 기반 앙상블은 시도하지 않은 방향  
**구현 난이도**: ★★☆☆☆

---

### C3. 스태킹 앙상블 (Stacking)

**설명**  
RF, XGB, LGB의 valid set 예측 확률을 2차 모델(meta-learner)의 입력으로 사용한다. 각 모델이 잡는 패턴이 다를 때 상호 보완 효과를 기대한다.

**이론적 근거**  
- Wolpert (1992): *Stacked Generalization.* Neural Networks — 스태킹의 이론적 기반
- Ting & Witten (1999): 메타 학습기가 base learner의 편향을 보정

**구현 방법**
```python
# Level-0: base learners
rf_proba  = rf.predict_proba(X_valid)[:, 1]
xgb_proba = xgb.predict_proba(X_valid)[:, 1]
lgb_proba = lgb.predict_proba(X_valid)[:, 1]

# Level-1: meta-learner
X_meta = np.column_stack([rf_proba, xgb_proba, lgb_proba])
meta = LogisticRegression().fit(X_meta, y_valid)
```

**주의**: valid 셋 양성이 39개로 극히 적어 meta-learner 학습이 불안정할 수 있음. Out-of-fold 예측 방식 권장.

**기대 효과**: ★★★☆☆ — RF가 XGB/LGB보다 훨씬 강한 상황에서 추가 이득이 제한적일 수 있음  
**구현 난이도**: ★★★☆☆

어텐션 기반 표형 데이터 딥러닝 모델. 각 스텝에서 중요한 피처를 선택적으로 attention하여 interpretable한 피처 중요도를 제공한다.

**이론적 근거**  
- Arik & Pfister (2021): *TabNet: Attentive Interpretable Tabular Learning.* AAAI — 여러 tabular 데이터셋에서 트리 기반 모델과 경쟁하는 성능. 피처 수가 많고 비선형 상호작용이 복잡할 때 우위

**주의**: 소표본(train 양성 283개)에서 딥러닝은 과적합 위험이 높음. 데이터 추가 수집 이후 시도 권장.

**기대 효과**: ★★☆☆☆ — 현재 데이터 규모에서는 효과 불확실. 데이터 수집 후 재검토

## D. 문제 프레이밍 변경

### D1. 생존 분석 (Cox Proportional Hazards Model)

**설명**  
"상폐 여부(0/1)"가 아닌 "상폐까지 남은 시간"을 모델링한다. 검열(censored) 데이터(아직 상폐되지 않은 기업)를 정보 손실 없이 활용할 수 있다.

**이론적 근거**  
- Cox (1972): *Regression Models and Life Tables.* JRSSB — Cox PH 모델의 원전. 의학에서 시작해 금융 부실 예측에 광범위 적용
- Lane, Looney & Wansley (1986): *An Application of the Cox Proportional Hazards Model to Bank Failure.* JBF — 은행 부도 예측에 Cox 모델 적용, 로지스틱 회귀보다 우수
- Shumway (2001): *Forecasting Bankruptcy More Accurately: A Simple Hazard Model.* JB — 이산시간 hazard model(= 기간별 로지스틱 회귀)이 Altman Z-score보다 정확
- 기업 패널 데이터는 생존 분석의 이상적 적용 대상 — 각 기업이 "상폐 시점"이라는 이벤트를 가짐

**구현 방법**
```python
from lifelines import CoxPHFitter, CoxTimeVaryingFitter

# 시간가변 공변량(time-varying covariates) 모형
# 각 행: (stock_code, start_year, stop_year, event=상폐여부, X1, X2, ...)
ctv = CoxTimeVaryingFitter()
ctv.fit(df_panel, id_col="stock_code", event_col="delisted",
        start_col="start_year", stop_col="stop_year")
```

**장점**  
- 상폐되지 않은 기업들의 연도별 데이터를 negative로 버리지 않고 정보로 활용
- 특정 연도가 아닌 "위험 함수 (hazard function)"로 시간에 따른 위험 변화를 모델링
- fixed_N1 vs N2 vs N3 선택 불필요 — 전체 패널을 하나로 모델링

**기대 효과**: ★★★★☆ — 프레이밍 자체를 바꾸는 접근. 패널 데이터 구조를 완전히 활용  
**구현 난이도**: ★★★☆☆

---

### D2. 다중 시계열 통합 라벨링 (N1 + N2 + N3)

**설명**  
fixed_N1은 양성 283개, N2는 293개, N3는 316개다. 세 가지를 별개로 학습하는 대신, 단계별 위험도를 다중 레이블로 통합한다.

**이론적 근거**  
- Ordinal classification: 상폐까지의 거리를 순서형 변수로 취급 (N3=위험 초기, N2=위험 중간, N1=위험 임박)
- Franses & Paap (2001): 순서형 로짓 모형이 이분 분류보다 더 많은 정보를 활용

**구현 방법**
```
label 정의:
  0 = 정상 (상폐 무관)
  1 = 주의 (상폐 3년 전, fixed_N3 해당)
  2 = 경계 (상폐 2년 전, fixed_N2 해당)
  3 = 위험 (상폐 1년 전, fixed_N1 해당)
```
→ 다중 클래스 분류 또는 순서형 회귀로 학습  
→ 최종 경보는 label≥2 또는 label≥3 threshold로 운영

**기대 효과**: ★★★☆☆ — 양성 샘플이 늘고(883개 통합) 모델이 위험 심화 과정을 학습  
**구현 난이도**: ★★★☆☆

---

### D3. 랭킹 문제로 재정의 (Learning to Rank)

**설명**  
"상폐 여부 분류"가 아닌 "위험 순위 매기기"로 목표를 바꾼다. PR-AUC는 사실상 ranking 능력 측정이므로, ranking 손실함수(LambdaRank, ListNet)로 직접 최적화하면 더 효과적일 수 있다.

**이론적 근거**  
- Burges et al. (2005): *Learning to Rank Using Gradient Descent.* ICML — LambdaRank의 원전
- Yan et al. (2003): AUC 최적화를 손실함수로 직접 사용 → 분류 손실보다 ranking task에 적합

**구현 방법**
```python
# XGBoost rank:pairwise 또는 LightGBM lambdarank
import lightgbm as lgb
params = {
    "objective": "lambdarank",
    "metric": "ndcg",
    "ndcg_eval_at": [5, 10],
    "label_gain": [0, 1, 2, 3],  # 위험도 가중치
}
```

**기대 효과**: ★★★☆☆ — 평가 지표(PR-AUC)와 학습 목표가 직접 연결  
**구현 난이도**: ★★★★☆

## E. 평가 체계 개선

### E1. Walk-Forward Cross-Validation

**설명**  
현재 valid=2023, test=2024 단일 분할은 valid 양성 39개로 threshold 최적화가 불안정하다. Walk-forward CV로 여러 연도를 순차적으로 검증하면 평가 안정성이 높아진다.

**이론적 근거**  
- Bergmeir & Benitez (2012): 시계열 데이터에서 전통적 k-fold는 미래 정보 유입 위험 — walk-forward가 표준
- Arlot & Celisse (2010): cross-validation의 이론적 성질

**구현 예시**
```
Fold 1: train=2015~2019 / valid=2020
Fold 2: train=2015~2020 / valid=2021
Fold 3: train=2015~2021 / valid=2022
Fold 4: train=2015~2022 / valid=2023  ← 현행
Test  : 2024 (단 한 번)
```
4개 valid fold의 PR-AUC 평균으로 하이퍼파라미터 선택 → 현재 39개 불안정 문제 완화.

**기대 효과**: ★★★☆☆ — 모델 성능 자체보다 **평가 신뢰도** 향상. 올바른 하이퍼파라미터 선택으로 간접적 성능 향상  
**구현 난이도**: ★★★☆☆
